Conceptual Questions

  1. Describe how parameter estimation in Gradient Descent differs from using the normal equation.

Gradient Descent comprises parameters like Theta, Learning rate as opposed to the coefficients and intercept used for linear regression. It is more of an optimisation method compared to normal equation. Normal equation would only work on linear data whereas GD works on linear as well as non-linear data. Unlike Gradient Descent, the normal equation provides the global minimum of the cost function for linear regression problems and it is also computationally expensive for large datasets.

  1. We used Mini-Batch Gradient descent here, but there are a variety of options for Gradient Descent. How is Mini-Batch Gradient Descent different from Stochastic Gradient Descent?

Stochastic GD is basically Mini-batch of size 1. because of batch size being one it is very unstable since in the case of an outlier encountered on the way it would lead to more noise and probably overshoot thus increasing the time for it to reach the local minima. In contrast, doing it for a batch can be much more stable as the batching makes it less noisy. Parallel processing of batch GD also makes it more favorable more computational purposes.   1. Identify your best-performing version of Gradient Descent and compare the error metrics for test data vs training data. Does this version of gradient descent show signs of overfitting? Explain.

The most effective model we've found utilizes a mini-batch size of 2000. It exhibits the lowest error on the test set and achieves the highest explained variance and R2 values. Importantly, it doesn't appear to suffer from overfitting, as indicated by the fact that the root mean squared error (RMSE) on the training set is only slightly lower and remains comparable to that of the test set. This suggests that the model generalizes well beyond the training data.

  1. Explain what the learning rate is for. Even though we did not, why might it be helpful to change the learning rate over time?

The learning rate plays a crucial role in determining how quickly the algorithm converges to the minimum of the cost function. It essentially controls the step size taken in each iteration of the gradient descent process.The learning rate would depend on the slope of the curve and also the cost function. If we see that our cost function is somehow increasing instead of decreasing, we need to increase the learning rate and vice versa.

  1. Why would a data scientist want to use early stopping in gradient descent?

Early stopping means stopping gradient descent when the validation error starts to increase. This indicates that it is applied to prevent overfitting. It also streamlines the model selection process by providing a clear criterion for determining the optimal stopping point
